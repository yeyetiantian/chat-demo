# 离线迁移开发环境 — 完整步骤

## 场景

```
电脑 A（有网络）            电脑 B（只有 Python 3.10，无网络）
  ├── 完整的项目代码          ├── 没有 venv
  ├── 已有 venv + 依赖       ├── 需要装依赖
  └── 可以跑 pip install     └── 不连外网
```

## 步骤

### 第 1 步：电脑 A 上导出依赖包

```bash
# 1. 进入项目根目录
cd /Users/wanglei/Desktop/ChatDemo

# 2. 创建离线包目录
mkdir -p pip_offline

# 3. 下载所有后端依赖（包含 PyInstaller）
cd backend
./venv/bin/pip download -r requirements.txt \
  -d ../pip_offline \
  --platform manylinux2014_x86_64 \
  --python-version 3.10 \
  --only-binary cryptography

# 4. 下载 databao-agent（Chat Demo 核心）
./venv/bin/pip download databao-agent \
  -d ../pip_offline \
  --python-version 3.10 \
  --only-binary cryptography

# 5. 下载 PyInstaller（如果要打包）
./venv/bin/pip download pyinstaller -d ../pip_offline

# 6. 回到项目根目录
cd ..
```

### 第 2 步：打包传输文件

```bash
# 1. 把整个项目 + 离线包打成一个压缩包
tar -czf chatdemo_offline.tar.gz \
  . \
  --exclude=node_modules \
  --exclude=__pycache__ \
  --exclude=.git \
  --exclude=*.pyc \
  --exclude=dist \
  --exclude=backend/logs/traces \
  --exclude=backend/logs/charts \
  --exclude=frontend/dist

# 2. 拷到 U 盘
cp chatdemo_offline.tar.gz /Volumes/U盘/
```

### 第 3 步：电脑 B 上还原环境

```bash
# 1. 解压到目标目录
tar -xzf chatdemo_offline.tar.gz -C ~/ChatDemo
cd ~/ChatDemo

# 2. 创建虚拟环境
python3.10 -m venv backend/venv

# 3. 离线安装依赖
cd backend

# 先装核心依赖
./venv/bin/pip install --no-index \
  --find-links ../pip_offline \
  -r requirements.txt

# 再装 databao-agent
./venv/bin/pip install --no-index \
  --find-links ../pip_offline \
  databao-agent --only-binary cryptography

# 4. 验证安装
./venv/bin/python -c "import duckdb; import fastapi; import databao; print('✅ 依赖安装成功')"
```

### 第 4 步：启动

```bash
# 后端
cd ~/ChatDemo && backend/venv/bin/python backend/run.py

# 前端（另一个终端）
cd ~/ChatDemo/frontend && npm run dev
```

## 常见问题

### Q1：pip download 说平台不兼容？

目标电脑是 Linux，但你在 Mac 上下载，需要指定平台参数：

```bash
# 如果目标电脑是 Linux x64
--platform manylinux2014_x86_64

# 如果目标电脑是 macOS x64
--platform macosx_10_9_x86_64

# 如果目标电脑是 Windows x64
--platform win_amd64
```

也可以不加 `--platform`，直接下载所有平台的包：

```bash
./venv/bin/pip download -r requirements.txt -d ../pip_offline
```

这样 `pip_offline/` 里会包含各平台的 whl，在目标电脑上安装时 pip 会自动选合适的。

### Q2：cryptography 编译报错？

```bash
# 已经在命令里加了 --only-binary cryptography
# 如果还报错，说明目标电脑缺少编译工具
# 方案：在 pip_offline 目录确认有 cryptography 的 whl
ls ../pip_offline/cryptography*.whl
```

### Q3：目标电脑没有 Node.js？

前端开发需要 Node.js 来跑 `npm run dev`。如果目标电脑没有 Node.js：

```bash
# 方案 A：在电脑 A 上先 build 好前端
cd frontend && npm run build

# 然后电脑 B 上不需要 Node.js，直接用后端托管静态文件
# 电脑 B 只需要启动后端：
backend/venv/bin/python backend/run.py

# 浏览器访问 http://localhost:8080 即可
# 不需要额外启动前端 dev server
```

### Q4：pip download 时部分包下载失败？

```bash
# 重试单个包
./venv/bin/pip download databao-agent -d ../pip_offline

# 查看已下载的包
ls ../pip_offline/*.whl | wc -l
```

### Q5：目标电脑的 Python 版本不一致？

本项目开发时用 Python 3.11，目标电脑是 3.10，理论上兼容。但不排除某些包需要特定版本。

```bash
# 如果 3.10 装不上，可以看具体报错
# 大部分包（duckdb、fastapi、databao）都支持 3.10
```

## 推荐方案

**方案一（exe）比方案二简单 10 倍。**

| 对比 | 方案一：exe | 方案二：离线 pip |
|------|------------|----------------|
| 目标电脑需要 | 无 | Python 3.10 |
| 传输大小 | ~80MB | ~200MB + 项目代码 |
| 步骤数 | 2 步（拷→跑） | 10+ 步 |
| 前端开发 | 不能改代码 | 可改可调试 |
| 推荐场景 | 给用户/测试用 | 开发者继续开发 |
