# DataPivot 项目路线图

## 一、当前状态

### ✅ 已完成功能

| 模块 | 功能 | 完成时间 |
|------|------|---------|
| **AI 对话分析** | 自然语言查询 → SQL → 图表，SSE 流式推送、多轮记忆、追问建议 | ✅ |
| **链路追踪** | AI 思考过程全透明：LLM 调用链、SQL 执行、耗时统计 | ✅ |
| **拖拽透视** | 字段面板 + 行/列/值拖拽配置、6 种图表切换、透视表 | ✅ |
| **数据下钻** | 人员→任务→规则→报警→信号 5 层下钻、波形图 | ✅ |

## 二、改进方向

### Phase 0：基础建设（一周）

| 任务 | 说明 | 外部依赖 |
|------|------|---------|
| **图表仪表盘** | 图表保存/编辑/删除/复制、网格展示、缩略图渲染 | 无 |
| **聊天历史持久化** | 后端 JSON 文件存储，刷新/切换页面不丢 | 无 |
| **AI 与图表仪表盘联动** | AI 对话结果可直接编辑并关联到仪表盘配置 | 无 |
| **AI 一键生成看板** | 通过一句自然语言指令，AI 自动生成包含多个图表的完整看板 | 无 |
| **内部 LLM 适配** | 适配内部私有 LLM 接口，替换 DeepSeek/OpenAI | LLM 接口文档 |
| **测试环境运行验证** | 在测试环境中部署运行，验证完整链路可用性 | 测试服务器 |
| 仪表盘 Bug 修复 | 缩略图不渲染、行列混淆、新建状态未重置等 5 项 | 无 |

### Phase 1：体验优化（1-2 周 · 7月中旬）

| 任务 | 时间 | 说明 | 外部依赖 |
|------|------|------|---------|
| SSE 断线重连 | 第 1 周 | 生成中断网/刷新时，自动重试最后一条未完成的查询 | 无 |
| 图表批量加载优化 | 第 1 周 | 仪表盘多图表懒加载、缩略图缓存 | 无 |
| 新手指引 | 第 2 周 | 首次使用引导提示、示例问题一键发送 | 设计支持 |
| 搜索历史会话 | 第 2 周 | 按关键词搜索历史对话记录 | 无 |

### Phase 2：功能增强（2-4 周 · 7月下旬）

| 任务 | 时间 | 说明 | 外部依赖 |
|------|------|------|---------|
| 图表导出 PNG/PDF | 第 3 周 | 图表右键导出图片，仪表盘导出报告 | 无 |
| 多数据源接入 | 第 3-4 周 | 支持 PostgreSQL、MySQL 等外部数据库连接 | 数据库选型 |
| 数据缓存 | 第 4 周 | 高频查询结果缓存加速 | 无 |
| 对话线程过期 | 第 4 周 | 自动清理 7 天前的历史会话 | 无 |

### Phase 3：vcloud 对接（4-8 周 · 8月-9月）

| 任务 | 时间 | 说明 | 外部依赖 |
|------|------|------|---------|
| vcloud 用户体系对接 | 第 5-6 周 | 接入 vcloud 统一登录与权限体系 | vcloud 认证接口 |
| vcloud 数据源对接 | 第 6-7 周 | 从 vcloud 平台获取业务数据，替换本地 DuckDB | vcloud 数据 API |
| 报表推送至 vcloud | 第 7-8 周 | 将分析结果/看板以报表形式推送到 vcloud 平台 | vcloud 消息通道 |
| 嵌入式集成 | 第 8 周 | 以 iframe 或微前端方式嵌入 vcloud 平台 | 前端架构支持 |

## 三、外部依赖

| 依赖项 | 用途 | 建议方案 | 优先级 |
|--------|------|---------|--------|
| **LLM API Key** | AI 对话核心功能 | DeepSeek API（¥0.14/百万 token，约 0.1 分/次查询） | 🔴 必须 |
| **数据库** | 业务数据存储 | 当前 DuckDB 单文件，可扩展至 PostgreSQL | 🟡 后续 |
| **UI/UX 设计** | 新手引导、图表美化 | 设计资源支持 | 🟡 后续 |
| **Docker 环境** | Web 部署模式 | 开发团队镜像仓库 | 🟢 长期 |
| **消息通道** | 定时报表推送 | 企业微信/钉钉/邮件 API | 🟢 长期 |

## 四、上线计划

### 阶段 1：内部试用（当前 · 7月）

```
形式:   GitHub Release 下载 exe
范围:   全部核心功能 + 仪表盘 + 聊天历史
用户:   团队内部 5-10 人
目标:   收集反馈，修复问题
准入:   ✅ 已完成
```

### 阶段 2：小范围发布（2 周后 · 7月中旬）

```
形式:    Release 发布 + 语义化版本号
范围:    Phase 1 优化完成 + 基础功能稳定
用户:    合作团队/早期用户
指标:    AI 查询成功率 > 95%，无严重 Bug
准入:    Phase 1 体验优化完成
```

### 阶段 3：正式发布（1-2 个月后 · 8-9月）

```
形式:    提供两种部署形态
         1. 单文件 exe — 个人用户，双击即用
         2. Docker 镜像 — 团队部署，Web 访问
范围:    全部功能 + 用户权限 + Web 部署
准入:    Phase 2 + Phase 3 完成
```
