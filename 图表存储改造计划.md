# 图表存储与配置系统 — 改造计划

## 一、需求分析

### 现状

```
AI 对话 (chat.py)                    Pivot 页面 (PivotPage.vue)
  │                                       │
  │ 生成 chart_spec + data                │ 拖拽字段 → 执行查询 → 渲染
  │ 结果展示完就丢了                       │ 每次重新配置，无法保存
  │                                       │
  └────────── 两套体系，互不相通 ───────────┘
```

### 目标

```
AI 对话                              配置页面
  │                                       │
  │ 生成 chart_spec + data                │ 加载已保存的图表
  │ → "保存到仪表盘"                       │ → 批量显示
  │ → 可编辑/调整                         │ → 可编辑/删除
  │                                       │
  └─────── 统一的数据结构，双向互通 ───────┘
                    │
                    ▼
          本地 JSON 文件存储
          (同 trace 机制)
```

## 二、数据模型设计

### 统一图表配置 JSON 结构

这是 AI 输出和 Pivot 配置之间的**对齐桥梁**：

```json
{
  "chart_id": "ch_a1b2c3d4",
  "name": "各车型报警数量统计",
  "description": "按车型分组统计报警总数，柱状图展示",
  "created_at": "2026-06-29T14:30:00",
  "updated_at": "2026-06-29T15:00:00",
  "source": "ai",              // "ai" | "manual" | "edited"
  "source_query": "按车型统计报警数量",  // AI 的原始问题（AI 来源时）
  "chart_type": "bar",         // bar | pie | line | waveform | radar | scatter
  "chart_spec": {              // Vega-Lite 完整 spec（AI 生成或手动编辑）
    "$schema": "https://vega.github.io/schema/vega-lite/v5.json",
    "mark": "bar",
    "encoding": {
      "x": {"field": "车型", "type": "nominal"},
      "y": {"field": "报警数量", "type": "quantitative"}
    }
  },
  "data_config": {             // 数据源配置（Pivot 配置用，AI 来源可为空）
    "dimensions": ["车型"],
    "measures": ["报警数量"],
    "aggregations": ["count"],
    "filters": {}
  },
  "data": [                    // 实际数据行（AI 来源时直接保存，Pivot 配置时执行查询后填充）
    {"车型": "货车", "报警数量": 145},
    {"车型": "客车", "报警数量": 89}
  ],
  "columns": ["车型", "报警数量"]
}
```

### 关键对齐点

| 字段 | AI 输出时 | Pivot 配置时 |
|------|----------|-------------|
| `chart_spec` | AI 生成的 Vega-Lite JSON | 用户可编辑/覆盖 |
| `data_config` | 可从查询中推断 | 用户拖拽配置（行/列/值/聚合） |
| `data` | AI 查询到的实际数据 | 为空 → 加载时按 `data_config` 查数据库 |
| `source` | `"ai"` | `"manual"` 或 `"edited"` |

**双向转换逻辑：**

```
AI 输出 → 统一 JSON
  chart_spec: AI 生成的 Vega-Lite ✔
  data_config: 从 AI 的 SQL 结果列推断（维度/度量分类）
  data: 实际结果行 ✔

Pivot 配置 → 统一 JSON
  chart_spec: 根据 chart_type + data_config 前端或后端构建
  data_config: 用户拖拽的字段 ✔
  data: 按 data_config 实时查询数据库后填充
```

## 七、数据结构对齐详述

### AI 输出 → 统一 JSON 的映射

```python
# chat.py 中保存时的转换逻辑
def ai_result_to_chart_config(ai_result: dict) -> dict:
    return {
        "chart_id": f"ch_{uuid.uuid4().hex[:8]}",
        "name": ai_result["query"][:30] + "...",
        "source": "ai",
        "source_query": ai_result["query"],
        "chart_type": ai_result["chart_type"],
        "chart_spec": ai_result["data"]["chart_spec"],
        "data_config": {
            "dimensions": ai_result["data"]["dimensions"] or [],
            "measures": ai_result["data"]["measures"] or [],
            "aggregations": ["count"],
            "filters": {}
        },
        "data": ai_result["data"]["data"],
        "columns": ai_result["data"]["columns"],
    }
```

### Pivot 配置 → 统一 JSON 的映射

```python
# Pivot 配置保存时的转换逻辑
def pivot_config_to_chart_config(pivot_config: dict) -> dict:
    chart_type = pivot_config.get("chart_type", "bar")
    return {
        "chart_id": f"ch_{uuid.uuid4().hex[:8]}",
        "name": pivot_config.get("name", f"图表 {日期}"),
        "source": "manual",
        "chart_type": chart_type,
        "chart_spec": build_spec(chart_type, pivot_config),  # 根据字段构建Vega-Lite
        "data_config": {
            "dimensions": pivot_config.get("dimensions", []),
            "measures": pivot_config.get("measures", []),
            "aggregations": pivot_config.get("aggregations", ["count"]),
            "filters": pivot_config.get("filters", {})
        },
        "data": [],  # 加载时按 data_config 实时查询
        "columns": [],
    }
```

### 加载时数据填充

```python
# 按 data_config 重新查询数据库
def refresh_chart_data(chart_config: dict) -> dict:
    if chart_config["data_config"]["dimensions"] or chart_config["data_config"]["measures"]:
        # 有 Pivot 配置 → 调用 DuckDBService 的聚合查询
        result = duckdb_service.aggregate(
            rows=chart_config["data_config"]["dimensions"],
            columns=[],
            values=chart_config["data_config"]["measures"],
            aggregations=chart_config["data_config"]["aggregations"],
            filters=chart_config["data_config"].get("filters", {}),
        )
        chart_config["data"] = result["rows"]
        chart_config["columns"] = result["columns"]
    # 如果没有 data_config（纯 AI 生成），data 已在保存时固化
    return chart_config
```

## 八、风险与注意事项

### 数据一致性

- AI 生成的图表保存后，如果数据库更新了，图表数据是**旧的**（AI 查询时的快照）
- 需要提供"刷新数据"按钮，按 data_config 重新查询
- 纯 AI 生成的图表（无 data_config）无法自动刷新，需用户补充 data_config

### 存储性能

- 图表数据量大时，JSON 文件可能变大（超过几百 KB）
- 方案：data 字段超过 1000 行时截断，只存摘要，加载时按需查询
- 索引文件保持轻量（只存摘要信息）

### 前后端对齐

- AI 的 `chart_spec` 是完整的 Vega-Lite JSON
- Pivot 的 `chart_spec` 需要根据字段和图表类型构建
- `ChartRenderer.vue` 已经同时支持这两种方式，不需要修改

### Vega-Lite Spec 校验

- 用户编辑 Vega-Lite JSON 时可能语法错误
- 需要做 JSON schema 校验或 try-catch 渲染
- ChartRenderer 已有错误显示逻辑
